# HI-REMS-SERVER
---

### 로컬 개발 환경 구축


#### 1. 필수 소프트웨어 설치
* **Node.js**: [공식 다운로드](https://nodejs.org/ko/download) (LTS 버전 권장)

#### 2. 패키지 설치 및 환경 설정
```bash
npm install
```
#### 3. 환경 변수 설정
```bash
# .env.example 파일을 복사하여 .env 파일을 생성하고 로컬 DB 접속 정보를 수정
cp .env.example .env
```
#### 4. 데이터베이스 초기화 및 스키마 추가

```sql
CREATE DATABASE alliothub;

# ... 필요한 테이블들 추가( 운영 환경에 맞게 추가 (members, imei_meta 등))

CREATE EXTENSION IF NOT EXISTS timescaledb CASCADE;
```

#### 5. 서버 실행
```bash
node src/app.js
```

#### 2. 스크립트 실행

```bash
node scripts/seed-gs-data.js
```

#### 3. 데이터 설명 및 추가 방법

#### [Body 데이터 예시 분석]
* **Raw Data**: `14 01 01 00 00 01 5e 00 32 04 e2 00 dc 00 1a 04 e2 03 e7 02 58 00 00 00 00 00 0f 42 40 00 00`

| 커맨드 | 에너지원 | 타입 | 멀티 | 에러코드 | 계측 및 누적 데이터 (26 Bytes) |
| :---: | :---: | :---: | :---: | :---: | :--- |
| 14 | 01 | 01 | 00 | 00 | 01 5e 00 32 04 e2 00 dc 00 1a 04 e2 03 e7 02 58 00 00 00 00 00 0f 42 40 00 00 |

**분석 결과:**
1. **에너지원**: 태양광 단상 (0x01, 0x01)
2. **상태**: 정상 가동 중 (Error Code 0x00)
3. **누적 발전량**: 1,000,000 Wh (Hex `0f 42 40` 변환 값)


- 데이터 전송 프로토콜 명세 (Body 구조)

| 위치 (Byte) | 필드명 | 크기 | 설명 |
| :--- | :--- | :--- | :--- |
| **1st** | **Command** | 1 Byte | 커맨드 |
| **2nd** | **Energy Source** | 1 Byte | 에너지원 구분 코드|
| **3rd** | **Type** | 1 Byte | 설비 타입 정보 |
| **4th** | **Multi** | 1 Byte | 멀티/단독운전 |
| **5th** | **Error Code** | 1 Byte | 장비 에러 상태 코드 |
| **6th ~ 31st** | **Data** | 26 Byte | 인버터 측정 데이터 및 상세 정보 |

- 상세 데이터 필드 정의 (6~31 Byte)

| 구간 (Byte) | 필드명 | 단위 | 설명 |
| :--- | :--- | :--- | :--- |
| **6 ~ 20** | **상세 계측 데이터** | V, A, W 등 | 전압, 전류, 주파수, 역률 등 인버터 정보 |
| **21 ~ 28** | **누적 발전량 (Wh)** | Wh | 현재까지의 총 누적 발전량 |
| **29 ~ 30** | **상태 플래그** | Bit | 고장 여부 |
| **31** | **종료 바이트** | - | 예비 필드 |

- 주의사항
- 기준 : 가장 최근 로그의 `time`이 현재 시각으로부터 90분 이상 경과화면 대시보드에서 해당 설비는 `오프라인`으로 표시됩니다.`
- 참고 : 더미데이터 사용 시 `body_logs.csv`의 시간 필드를 현재 시점에 가장 가깝게 수정하여 재삽입하십시오.
